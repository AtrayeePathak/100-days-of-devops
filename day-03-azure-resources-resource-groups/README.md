# Day 03 — Azure Resources, Resource Groups, and ARM

**Date:** 2026-07-08

## What I set out to learn

Understand how Azure organizes resources, how resource groups work, and why Azure Resource Manager is the control plane behind most deployments.

## What I did

- Reviewed the difference between Azure resources and resource groups
- Looked at how Azure Resource Manager (ARM) manages deployments and dependencies
- Practiced thinking about how resources relate to one another inside a single project
- Compared the idea of a resource group to a logical container for related services

## What I learned / key takeaways

- A resource group is a management boundary, not a networking or security boundary by itself
- Azure Resource Manager gives a consistent way to create, update, and delete resources
- Organizing resources into meaningful groups makes cleanup, permissions, and lifecycle management much easier
- A good Azure project structure starts with clear resource grouping and naming
