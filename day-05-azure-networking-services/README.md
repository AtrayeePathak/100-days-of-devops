# Day 05 — Azure Networking Services

**Date:** 2026-07-10

## What I set out to learn

Build a stronger mental model of Azure networking: VNets, subnets, route tables, NSGs, and application security groups.

## What I did

- Reviewed the difference between a VNet and a subnet
- Studied how route tables direct traffic inside Azure networks
- Looked at NSGs as a first layer of network filtering
- Explored how ASGs help organize access rules for related workloads

## What I learned / key takeaways

- A VNet is the core networking boundary for Azure resources
- Subnets help divide networks into logical segments for security and organization
- NSGs and ASGs are both useful, but they solve different problems in network design
- Good networking design begins with clear segmentation and controlled access paths
