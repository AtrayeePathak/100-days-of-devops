# Day 13 — Azure DevOps in 20 Minutes

**Date:** 2026-07-22

## What I set out to learn

Understand what Azure DevOps is, why DevOps engineers use it, and how it helps teams with planning, collaboration, automation, and delivery.

## What I did

- Watched a short overview of Azure DevOps and its core purpose in modern software delivery
- Learned about the main Azure DevOps services: Boards, Repos, Pipelines, Test Plans, and Artifacts
- Explored how DevOps engineers use Azure DevOps to manage work items, version control, CI/CD, and releases
- Reviewed the difference between Azure DevOps and GitHub in a practical team workflow context
- Noted how Azure DevOps supports end-to-end delivery from idea to deployment

## What I learned / key takeaways

- Azure DevOps is a platform for planning, coding, building, testing, and deploying software
- DevOps engineers use it to create visible workflows for teams and automate repetitive tasks
- Boards help organize tasks, user stories, and sprints
- Repos provide source control and team collaboration
- Pipelines enable continuous integration and continuous delivery with automation
- Artifacts help manage package feeds, which is useful for shared dependencies and deployments
- Azure DevOps is especially useful when teams need governance, traceability, and structured delivery processes
