# Day 04 — Azure Virtual Machines

**Date:** 2026-07-09

## What I set out to learn

Learn the basics of Azure virtual machines, including how they are created, connected to, and used for real workloads.

## What I did

- Reviewed the Azure VM creation flow from the portal
- Looked at key VM concepts such as size, disks, OS image, and networking
- Compared virtual machines with containers and serverless services
- Studied how a VM can host an application and why it remains a common workload choice

## What I learned / key takeaways

- Azure VMs are a flexible way to run custom workloads when full control is needed
- VM sizing and disk choice have a direct impact on performance and cost
- Security and access depend heavily on network configuration, SSH/RDP access, and role-based permissions
- Virtual machines are still very relevant in hybrid and lift-and-shift scenarios
