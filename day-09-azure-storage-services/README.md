# Day 09 — Azure Storage Services

**Date:** 2026-07-14

## What I set out to learn

Understand the main Azure storage services and when each one is a good fit for real workloads.

## What I did

- Reviewed Azure Storage accounts and their role as the base for many Azure data services
- Looked at common storage types such as Blob Storage, File Storage, Queue Storage, and Table Storage
- Studied the difference between hot, cool, and archive access tiers
- Compared storage choices based on access patterns, durability, and cost expectations

## What I learned / key takeaways

- Azure storage is not just one service; it is a family of services meant for different workloads
- Blob Storage is especially useful for unstructured data like files, images, and backups
- Storage choice should be driven by access frequency, data size, and performance needs
- Managed storage services often simplify application design compared to self-hosted alternatives
