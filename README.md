possible, a runnable example.

a DevOps/SRE role. No fluff commits — every entry below links to real work.

## How I use this repo

1. Copy `template-day/` → rename to `day-XX-topic-name/`
2. Spend 20–60 minutes learning + doing something small and concrete
3. Write up what I learned in that day's `README.md`
4. Commit and update the table above

No day needs to be huge. Small and consistent beats big and sporadic.
